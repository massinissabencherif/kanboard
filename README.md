# kanboard
Résumé du Projet Annuel - Kanboard
Objectif du Projet
L'objectif est de développer Kanboard, une application web de gestion de projet inspirée du système Kanban. Elle doit permettre aux utilisateurs de créer et gérer des projets collaboratifs en ajoutant et organisant des tâches sous forme de cartes.

Principales Fonctionnalités (15 points)
Visibilité (1 pt)
L’application doit être déployée sur Internet avec un nom de domaine sécurisé et un bon référencement.
Gestion des Projets (5 pts)
Les utilisateurs peuvent s'inscrire, se connecter et récupérer leur mot de passe via email.
Un utilisateur peut créer un projet et inviter des membres par email.
Les projets contiennent des tâches assignables aux membres.
Tâches (2 pts)
Une tâche contient : titre, description (optionnelle), catégorie (colonne Kanban), priorité, dates de création et d’échéance.
Possibilité d'organiser les tâches en colonnes (ex: "À faire", "En cours", "Fait").
Affichage des Tâches (2 pts)
Vue Kanban avec glisser-déposer des tâches.
Vue Liste pour filtrer et rechercher des tâches.
Vue Calendrier affichant les tâches selon leur échéance.
Design & Intégration Web (1 pt)
Application mobile-friendly, responsive et supportant les thèmes clair/sombre.
Technologies Utilisées (2 pts)
Front-end : HTML, CSS, JavaScript
Back-end : Laravel (PHP)
Base de données : SQL
Historisation & Conteneurisation (4 pts)
Utilisation de Git avec commits clairs et contributions suivies.
Le projet doit être conteneurisé (Docker) et réinstallable facilement.
Bonus (5 points)
Mode Hors-Ligne (2 pts)
Modification temporaire des tâches, synchronisation dès le retour en ligne.
Temps Réel (1 pt)
Mise à jour instantanée des modifications avec notifications (ex: Laravel Pusher).
Statistiques & Rapports (1 pt)
Analyse des tâches : taux d'achèvement, répartition par catégorie, productivité des membres.
Export iCal (1 pt)
Permettre la synchronisation avec Google Calendar, Outlook, etc.
Contraintes & Livrables
Le projet doit être soumis sous format ZIP (pas de lien Git).
Chaque membre doit être capable d'expliquer le code.
Respect des bonnes pratiques de développement.
Présentation professionnelle lors de la soutenance.
Conclusion
Ce projet vise à développer une application web complète de gestion de projet avec une interface moderne, une gestion des utilisateurs et un système de tâches en Kanban. L’utilisation de technologies modernes (Laravel, SQL, Docker) est requise, avec des fonctionnalités avancées comme le mode hors-ligne et la synchronisation en temps réel.
